docker build https://github.com/loyurs/qkrun.git#master:alicloud_images/x86_64-unknown-linux-musl
docker build giturl#<main/master/branch>:folder



buildkit 需要containerd即可，可以采取，新开一个docker的策略，以这个docker的containerd作为构建容器的容器来构建rm 构建完成后自行销毁，销毁后将构建的以target的形式放置在宿主机即可


构建语法后缀	提交使用	使用的构建上下文
myrepo.git	refs/heads/master	/
myrepo.git#mytag	refs/tags/mytag	/
myrepo.git#mybranch	refs/heads/mybranch	/
myrepo.git#pull/42/head	refs/pull/42/head	/
myrepo.git#:myfolder	refs/heads/master	/myfolder
myrepo.git#master:myfolder	refs/heads/master	/myfolder
myrepo.git#mytag:myfolder	refs/tags/mytag	/myfolder
myrepo.git#mybranch:myfolder	refs/heads/mybranch	/myfolder

旧版的buildkit不支持子目录

./buildctl build --frontend gateway.v0 --opt source=docker/dockerfile --opt context=https://github.com/loyurs/qkrun.git#master:docker/rust_code_server/ --output type=image,name=docker.io/liloyu/yuxin520:vk
